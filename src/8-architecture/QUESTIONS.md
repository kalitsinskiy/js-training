**Evaluation Format:**
- Total questions: 10
- Students: 5 candidates
- Each student gets: 2 questions randomly
- Prepare answers for ALL 10 questions!

---

**Q1:** Explain the Single Responsibility Principle with a code example. How do you identify a violation?

**Q2:** What is the Open/Closed Principle? Show how it prevents modifying existing code when adding features.

**Q3:** Explain the Dependency Inversion Principle. What is "programming to an interface" and why does it matter for testing?

**Q4:** What is MVC? Draw or describe the flow of a request from browser to database and back.

**Q5:** What is Layered Architecture? What is the rule about dependency direction and why is it important?

**Q6:** What is the Dependency Rule in Clean Architecture? Why should Domain never import from Infrastructure?

**Q7:** Design a REST API for a blog (posts, comments, users). Show the URLs, HTTP methods, and status codes.

**Q8:** What is the difference between an Entity and a Value Object in DDD? Give an example of each.

**Q9:** What is a Repository pattern? What problem does it solve and how does it improve testability?

**Q10:** What is coupling and cohesion? What is "low coupling, high cohesion" and why is it a goal?

---

**Evaluation Criteria:**
- Can explain the principle without memorized definitions
- Can identify violations in code
- Can propose a better architecture
- Connects principles to real-world tradeoffs
