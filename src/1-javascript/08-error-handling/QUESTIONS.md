# Week 9: Error Handling - Evaluation Questions

**Evaluation Format:**
- Total questions: 10
- Students: 5 candidates
- Each student gets: 2 questions randomly
- Prepare answers for ALL 10 questions!

---

**Q1:** What is `try...catch...finally`? When does each block execute? Can you omit `catch` or `finally`?

**Q2:** What's the difference between a syntax error and a runtime error? Which one can `try...catch` handle?

**Q3:** What is the `error` object passed to `catch`? What properties does it have?

**Q4:** When should you use `throw`? What can you throw (not just Error objects)?

**Q5:** What are the built-in error types in JavaScript? Give an example of when each occurs naturally.

**Q6:** What is the purpose of `finally`? Give a real-world example where it's essential.

**Q7:** How do you create a custom error class? What are the benefits of custom errors?

**Q8:** How do you handle errors in async/await code? Compare it to error handling in Promise chains.

**Q9:** What is error propagation? How do you decide whether to handle an error locally or re-throw it?

**Q10:** What's the difference between `error.message`, `error.name`, and `error.stack`? When is each useful?

---

**Evaluation Criteria:**
- Clear explanation of concepts
- Ability to provide code examples
- Understanding of "why", not just "what"
- Real-world use cases
